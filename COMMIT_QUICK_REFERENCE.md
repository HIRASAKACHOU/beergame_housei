# 📝 コミット クイックリファレンス

## 次のバージョン予定
- **現行**: v0.8
- **次**: v0.9（予定）

## 今すぐコミット（テンプレート）

### 1️⃣ ステータス確認
```bash
git status
```

### 2️⃣ 変更をステージ
```bash
git add -A
```

### 3️⃣ コミット（日本語）
```bash
git commit -m "v0.9: 変更内容を記述

**追加機能:**
- 機能1
- 機能2

**改善:**
- 改善1

**ファイル修正:**
- game.js: 修正内容
- index.html: 修正内容
"
```

### 4️⃣ プッシュ
```bash
git push origin main
```

### 5️⃣ CHANGELOG.md を更新
- 新しいエントリを上部に追加
- 日付とバージョンを記録

## チェックリスト

毎回コミット前に確認：

- [ ] 変更内容が明確に説明されている
- [ ] 日本語で記述されている
- [ ] ファイル修正リストが正確
- [ ] `git diff` で変更を確認した
- [ ] ローカルでテスト済み
- [ ] CHANGELOG.md 更新予定を確認

## 緊急参考

**プッシュ前に変更を取り消したい場合：**
```bash
git reset --soft HEAD~1
```

**リモートに間違えてプッシュした場合：**
```bash
git revert HEAD
git push origin main
```
