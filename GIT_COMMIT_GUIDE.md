# Git 提交指南 - ビールゲーム

## 概要
このドキュメントは、ビールゲームプロジェクトでのGitコミットとバージョン管理の標準的な流れを定義しています。

## バージョン管理ポリシー

### 現在のバージョン
- **メジャーバージョン**: 0（固定）
- **マイナーバージョン**: 0-9の範囲（段階的に更新）
- **現行最新版**: v0.8

### バージョン番号のルール
1. **0.x 形式のみ使用**
   - 例：v0.1, v0.2, ..., v0.9
   - v1.0以上への更新は、プロジェクト管理者のみが決定

2. **マイナーバージョン更新のトリガー**
   - 新機能の追加
   - 重大なバグ修正
   - UI/UX改善
   - パフォーマンス最適化
   - アーキテクチャ改善

## コミットメッセージ形式

### 標準フォーマット
```
v0.X: 主要な変更内容（日本語）

**追加機能:**
- 機能1説明
- 機能2説明

**改善:**
- 改善1説明
- 改善2説明

**変更:**
- 変更1説明

**ファイル修正:**
- ファイル1: 変更内容
- ファイル2: 変更内容
```

### 実例
```
v0.8: AI システムのパラメータ化実装

**更新内容:**
- AI戦略の硬コード係数をパラメータ化
- defaultParams オブジェクトでAIパラメータを集中管理

**UI改進:**
- 積極型からcalm型（冷静型）に改名

**機能:**
- 将来の需要曲線変動に対応可能な設計

**ファイル修正:**
- index.html: 4つのAIドロップダウンを更新
- game.js: AIStrategy クラスを完全に再構築
```

## 提交手順

### ステップ1: 変更を確認
```bash
git status
git diff game.js
git diff index.html
```

### ステップ2: すべてをステージング
```bash
git add -A
```

### ステップ3: 日本語でコミット
```bash
git commit -m "v0.X: 変更内容

**詳細説明**
...
"
```

### ステップ4: リモートにプッシュ
```bash
git push origin main
```

### ステップ5: CHANGELOG.md を更新
- 新しい変更エントリを追加
- 日付とバージョンを記録

## 大版本更新（v1.0へ）

### 条件
- プロジェクト管理者のみが決定
- 明確な理由とドキュメント

### 実行方法
```bash
# タグを作成
git tag v1.0 -m "v1.0: メジャーバージョンリリース

**更新内容:**
...
"

# タグをプッシュ
git push origin v1.0
```

## 注意事項

✅ **必ず日本語で記述**
- コミットメッセージは日本語
- 技術用語は英語のまま

✅ **詳細を記入**
- 何が変わったのか
- どのファイルが影響を受けたのか
- 使用例やコード例を含める

✅ **CHANGELOG.md を同期**
- 毎回のコミット後、CHANGELOGを更新
- 将来の参照のため、履歴を保持

❌ **避けるべきこと**
- 英語混合メッセージ
- 不明確な説明
- CHANGELOGの更新忘れ

## 用語集

| 用語 | 説明 |
|------|------|
| コミット | ローカル履歴にコード変更を記録 |
| プッシュ | ローカルのコミットをリモートに送信 |
| マージ | 複数のブランチの変更を統合 |
| チェックアウト | 異なるブランチまたはコミットに切り替え |
| ステージング | コミット対象のファイルを指定 |
| CHANGELOG | バージョン変更履歴のドキュメント |
