# 🍺 ビールゲーム v1.0 - シングルプレイヤーモード

## 📋 プロジェクト概要

ビールゲームは、供給チェーン全体における情報格差による「ブルウィップ効果」を体験するための古典的なビジネスゲームです。

このプロジェクトは：
- ✅ **ブラウザベース**: インストール不要、HTML/CSS/JavaScriptのみ
- ✅ **新UIシステム**: ホームページ、モード選択、設定ページを備えた現代的なUI
- ✅ **シングルプレイ**: 1人で複数の役割をプレイ
- ✅ **多言語対応**: 日本語・中国語に対応
- ✅ **AI対応**: パニック型・安全型・冷静型の3つのAI戦略
- ✅ **アニメーション**: 視覚的に分かりやすいUI/UX

## 🎮 ゲームモード

### 🎯 クラシックモード
- 伝統的なビールゲーム体験
- 標準的なルール
- 20/30/40週から選択
- 4つの役割を選択可能

### 🎴 カードモード
- 特殊カード効果を使った新しい体験
- 戦略的要素追加
- より高い挑戦性
- *（近日公開予定）*

### ⚙️ カスタムモード
- パラメータ調整可能
- 20/30/40/50/60週から選択
- AI行動パターン設定
- 輸送遅延、入荷時間、生産時間など調整可能
- 在庫コスト、欠品コストのカスタマイズ

## 🎮 ゲームルール

### 供給チェーンの役割（4つ）

| 役割 | 説明 | 基本機能 |
|------|------|--------|
| **🏪 小売業者** | 顧客に製品を販売 | 需要を予測し発注 |
| **📦 二次卸売業者** | 小売業者に供給 | 在庫と需要を管理 |
| **🚚 一次卸売業者** | 二次卸売業者に供給 | 供給チェーン中盤 |
| **🏭 工場** | 最初の製品供給者 | 生産と供給を管理 |

### ゲーム流れ

1. **入荷**: 上流から商品を受け取る
2. **出荷**: 下流へ商品を送る
3. **発注**: 次の週のための発注数量を決定

### コスト計算

```
累積コスト = (在庫数 × 在庫コスト) + (欠品数 × 欠品コスト)
```

**目標**: 累積コストを最小化する

## 🚀 クイックスタート

### 1. ローカルサーバーを起動
```bash
python -m http.server 8000
```

### 2. ブラウザでアクセス
```
http://localhost:8000
```

### 3. ゲームをプレイ
1. **言語選択**: 日本語または中国語を選択
2. **ゲームモード選択**: クラシックモード・カードモード・カスタムモード
3. **役割選択**: 4つの役割から1つ選択
4. **AI設定**: 他の役割のAI戦略を設定
5. **パラメータ調整**: カスタムモードでは各種パラメータを調整可能

## 🤖 AI戦略の詳細

### パニック型（Panic）
- **特徴**: 需要変化に過剰反応
- **用途**: チェーンに混乱をもたらす

### 安全型（Safe）
- **特徴**: 固定の安全在庫を維持
- **用途**: 保守的な戦略

### 冷静型（Calm）
- **特徴**: 低在庫を追求
- **用途**: 効率重視
