# 🍺 ビールゲーム - シングルプレイヤーモード


## 📋 プロジェクト概要

ビールゲームは、供給チェーン全体における情報格差による「ブルウィップ効果」を体験するための古典的なビジネスゲームです。

このプロジェクトは：
- ✅ **ブラウザベース**: インストール不要、HTML/CSS/JavaScriptのみ
- ✅ **シングルプレイ**: 1人で複数の役割をプレイ
- ✅ **日本語対応**: 完全な日本語インターフェース
- ✅ **AI対応**: パニック型・安全型・冷静型の3つのAI戦略
- ✅ **アニメーション**: 視覚的に分かりやすいUI/UX

## 🎮 ゲームルール

### 供給チェーンの役割（4つ）

| 役割 | 説明 | 基本機能 |
|------|------|--------|
| **🏪 小売業者** | 顧客に製品を販売 | 需要を予測し発注 |
| **📦 二次卸売業者** | 小売業者に供給 | 在庫と需要を管理 |
| **📦 一次卸売業者** | 二次卸売業者に供給 | 供給チェーン中盤 |
| **🏭 工場** | 最初の製品供給者 | 生産と供給を管理 |

### ゲーム流れ

1. **入荷**: 上流から商品を受け取る
2. **出荷**: 下流へ商品を送る
3. **発注**: 次の週のための発注数量を決定

### コスト計算

```
累積コスト = (在庫数 × 在庫コスト) + (欠品数 × 欠品コスト)
```

**目標**: 累積コストを最小化する

## 🚀 クイックスタート

### 1. ファイルを開く
```bash
# ローカルHTTPサーバーを起動
python -m http.server 8000
```

### 2. ブラウザでアクセス
```
http://localhost:8000
```

### 3. ゲームをセットアップ
1. **総週数**: 20週 / 30週 / 40週 を選択
2. **自分の役割**: 4つの役割から1つ選択
3. **AI役割設定**: 
   - ランダム: 毎回異なる決定
   - パニック型: 需要に過剰反応
   - 安全型: 安全在庫を優先
   - 冷静型: 低在庫を追求

### 4. ゲーム開始
- 毎週、出荷と発注を決定
- コスト履歴を追跡
- 30週後にゲーム終了

## 🤖 AI戦略の詳細

### パニック型（Panic）
- **特徴**: 需要変化に過剰反応
- **係数**: demandMultiplier = 1.5
- **用途**: チェーンに混乱をもたらす

### 安全型（Safe）
- **特徴**: 固定の安全在庫を維持
- **係数**: safetyStock = 8
- **用途**: 保守的な戦略

### 冷静型（Calm）
- **特徴**: 低在庫を追求
- **係数**: demandMultiplier = 0.9
- **用途**: 効率重視
